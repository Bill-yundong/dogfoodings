## 1. 产品概述

MetaboFlow 是一款基于 SolidJS 的个人膳食代谢平衡分析平台，通过异步代谢动力学预测引擎分析餐后血糖波峰趋势，结合千万级食物离线索引（IndexedDB），实现用户端与专业分析终端间的语义对齐，支撑全生命周期跨场景膳食协同管理。

- 面向注重健康的个人用户及营养师/医生等专业人员，解决膳食记录碎片化、血糖波动不可预测、用户与专业人员信息不对称的核心痛点
- 目标成为个人代谢健康管理的基座工具，以精准的升糖预测与离线可用性构建差异化壁垒

## 2. 核心功能

### 2.1 用户角色

| 角色 | 注册方式 | 核心权限 |
|------|----------|----------|
| 个人用户 | 邮箱注册 | 膳食记录、血糖预测、营养分析、个人报告 |
| 专业分析师 | 邀请码注册 | 语义对齐仪表盘、批量用户监控、深度代谢报告 |

### 2.2 功能模块

1. **膳食控制台（用户端）**：膳食录入、实时营养计算、血糖曲线预测、食物搜索（离线千万级索引）
2. **代谢分析终端（专业端）**：语义对齐面板、多用户代谢概览、异常波峰告警、专业报告生成
3. **食物营养库**：千万级食物数据浏览器、营养素详情、升糖指数标签、离线同步状态

### 2.3 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|----------|
| 膳食控制台 | 膳食录入面板 | 快速添加食物、支持搜索自动补全、分量调整、餐次标记 |
| 膳食控制台 | 实时营养仪表盘 | 宏量营养素环形图、微量营养素进度条、每日目标达成率 |
| 膳食控制台 | 血糖预测曲线 | 基于代谢动力学引擎的餐后2h血糖模拟曲线、波峰标注、安全区间高亮 |
| 膳食控制台 | 膳食时间轴 | 按时间线展示全天膳食记录、支持编辑删除 |
| 代谢分析终端 | 语义对齐面板 | 用户端膳食数据与专业分析维度的映射关系可视化、对齐置信度指标 |
| 代谢分析终端 | 多用户代谢概览 | 卡片式用户列表、关键代谢指标排序、异常标记 |
| 代谢分析终端 | 波峰异常告警 | 超阈值血糖波峰自动标记、历史趋势对比、风险等级标注 |
| 代谢分析终端 | 专业报告生成 | 一键生成代谢分析PDF报告、自定义报告模板 |
| 食物营养库 | 食物搜索浏览器 | 千万级食物全文搜索、分类筛选、升糖指数标签过滤 |
| 食物营养库 | 食物详情卡片 | 营养素全景展示、GI/GL值、同类食物替代推荐 |

| 登录/注册 | 角色选择 | 个人用户/专业分析师角色分流注册 |

## 3. 核心流程

用户在膳食控制台录入食物后，系统通过 IndexedDB 离线索引实时检索营养数据，异步代谢动力学引擎根据食物GI值、分量、用户代谢参数计算餐后血糖曲线，波峰数据经语义对齐层转换为专业分析维度，在专业分析终端同步呈现。

```mermaid
flowchart TD
    "用户录入膳食" --> "IndexedDB食物检索"
    "IndexedDB食物检索" --> "营养素聚合计算"
    "营养素聚合计算" --> "代谢动力学引擎"
    "代谢动力学引擎" --> "血糖曲线预测"
    "血糖曲线预测" --> "波峰检测与标注"
    "波峰检测与标注" --> "语义对齐层"
    "语义对齐层" --> "用户端可视化"
    "语义对齐层" --> "专业分析终端"
    "专业分析终端" --> "异常告警"
    "专业分析终端" --> "报告生成"
```

## 4. 用户界面设计

### 4.1 设计风格

- **主色调**：深绿 (#0A2E1C) + 亮绿荧光 (#00FF88) 为代谢活力色，辅以暖琥珀 (#F5A623) 标注波峰告警
- **次色调**：暗色系背景 (#0D1117) 搭配半透明玻璃质感卡片
- **按钮风格**：圆角胶囊形，悬停时发光扩散动效，主按钮荧光绿底色
- **字体**：标题使用 Space Grotesk（几何感科技风），正文使用 DM Sans（清晰可读）
- **布局风格**：左侧导航栏 + 内容区网格布局，数据可视化区域占主导
- **图标风格**：线性图标搭配微光晕，统一 24px 网格

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI元素 |
|----------|----------|--------|
| 膳食控制台 | 膳食录入面板 | 搜索输入框（自动补全下拉）、食物卡片网格、分量滑块、餐次切换标签 |
| 膳食控制台 | 实时营养仪表盘 | 环形进度图（Canvas）、营养素条形图、数字翻牌动效 |

### 4.3 响应式策略

- 桌面优先设计，最小适配 1280px 宽度
- 平板端（768-1280px）：导航栏折叠为底部标签栏，卡片网格从3列切为2列
- 移动端（<768px）：单列堆叠布局，血糖曲线简化为迷你趋势线，搜索框全宽展开

### 4.4 3D场景指导

不适用 — 本项目以2D数据可视化为核心，不涉及3D场景。
