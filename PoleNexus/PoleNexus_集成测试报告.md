## 2. 核心业务场景定义

### 2.1 场景一：路政管理 - 灯杆节点监控

**设计预期（0-1阶段）：**
- 管理员可实时查看所有灯杆节点状态
- 支持按状态（在线/离线/告警）和区域筛选
- 能耗数据通过 ECharts 可视化展示
- 数据实时同步到照明控制模块

**优先级：** P0

---

### 2.2 场景二：照明控制 - 批量调光控制

**设计预期（0-1阶段）：**
- 管理员可选择单个或多个灯杆进行调光
- 支持手动设置亮度（0-100%）
- 支持一键应用节能模式
- 调光指令异步下发并记录历史
- 状态变化实时同步到路政管理模块

**优先级：** P0

---

### 2.3 场景三：照明控制 - 自动节能模式

**设计预期（0-1阶段）：**
- 支持开启/关闭自动节能模式
- 节能规则可配置（时间范围、亮度、优先级）
- 系统根据时间、环境光、交通密度自动计算亮度
- 定时循环执行节能逻辑

**优先级：** P1

---

### 2.4 场景四：工况日志 - 离线记录管理

**设计预期（0-1阶段）：**
- 所有操作自动记录日志
- 日志存储在 IndexedDB 中，支持万级数据
- 支持按级别（info/warning/error）和同步状态筛选
- 支持关键词搜索
- 支持手动同步和历史清理

**优先级：** P1

---

### 2.5 场景五：模块间实时数据同步

**设计预期（0-1阶段）：**
- 照明控制模块的调光操作 → 路政管理模块能耗图表更新
- 所有模块操作 → 工况日志模块实时显示
- 使用 BroadcastChannel 实现跨模块通信

**优先级：** P0

## 4. Bug 修复验证

### 4.1 已修复 Bug 列表

| Bug ID | 问题描述 | 修复前状态 | 修复后状态 |
|-------|---------|-----------|-----------|
| BUG-001 | 自动节能开关无法打开 | ❌ 开关点击无反应 | ✅ 正常切换 |
| BUG-002 | 批量调光指令无响应 | ❌ 点击按钮无效果 | ✅ 正常发送指令 |
| BUG-003 | 应用节能模式无响应 | ❌ 点击无效果 | ✅ 正常计算并应用 |

### 4.2 BUG-001 修复详情

**问题根因：**
模板中同时使用 `v-model` 和 `@change` 处理同一变量，导致：
1. v-model 自动更新值（false → true）
2. @change 回调中再次取反（true → false）
3. 最终值保持不变

**修复方案：**
```vue
<!-- 修复前 -->
<input type="checkbox" v-model="autoSavingEnabled" @change="toggleAutoSaving" />

<!-- 修复后 -->
<input type="checkbox" :checked="autoSavingEnabled" @change="toggleAutoSaving" />
```

**验证结果：**
- 开关可正常开启/关闭 ✅
- 开启时定时器正确启动 ✅
- 关闭时定时器正确清除 ✅

### 4.3 BUG-002/003 修复详情

**问题根因：**
1. 使用 `Set<string>` 存储选中状态，Vue 响应式不完整
2. 复选框使用 `:checked` + `@change` 而非 `v-model`
3. 缺少用户操作反馈

**修复方案：**
```typescript
// 修复前
const selectedPoles = ref<Set<string>>(new Set())

// 修复后
const selectedPoles = ref<Record<string, boolean>>({})
const selectedPoleIds = computed(() => 
  Object.entries(selectedPoles.value)
    .filter(([_id, selected]) => selected)
    .map(([id]) => id)
)
```

**修复点：**
1. Set → Record<string, boolean>：更好的响应式支持
2. 复选框：`:checked/@change` → `v-model`
3. 新增消息提示组件（showMessage）
4. 增强 try-catch 错误处理
5. 按钮禁用状态使用 computed 变量
